Click map reference
Document rail
What you click
What happens
1
"+" button
Opens file upload — drag or browse. Accepts PDF, XLSX, images. Auto-queues OCR + extraction on upload.
2
Document row
Selects the document. Center panel switches to Document mode showing the full content with findings highlighted inline. Right panel shows entities extracted from this document.
3
Finding count badge
Stays in Findings mode but filters the center list to show only findings from this document.
Center panel — Findings mode
What you click
What happens
4
"Findings" toggle
Stays in / returns to findings list. If a document was selected, shows all findings (not just that document's).
5
"Document" toggle
Switches center to Document mode. Shows the last-selected document, or prompts to pick one from the rail if none is selected.
6
"+ Add finding" button
Opens a drawer or modal. You write a title, description, pick severity and evidence weight, and optionally link documents and entities. Creates a manual finding (source = MANUAL).
7
Filter chips
Filters the finding list. Chips are: All / Critical / High / Medium / Confirmed / Needs Evidence / New / Dismissed / AI (AI-generated) / Rule (auto-detected). Multiple chips can be active.
8
Finding card
Selects the finding. Right panel updates to show entities, evidence snapshot, connected findings, and leads-to actions for this finding. Does NOT navigate away.
9
Document chip on a finding
Switches center to Document mode, opens that document, and scrolls to + highlights the passage relevant to this finding.
10
Status chip (Confirmed/New/etc)
Opens an inline status picker. One click to change status without opening a modal. Options: New / Needs Evidence / Confirmed / Dismissed. Dismissed asks for a reason note.
Right panel — finding selected
What you click
What happens
11
Entity row (person/org/property)
Navigates to the entity detail page. Shows all findings, documents, and external search results tied to that entity. Back button returns here.
12
Connected finding
Scrolls the center panel to that finding and selects it. Right panel updates to the new finding's context. No navigation — everything stays in place.
13
"Leads to" action card
If it's a "Next pull" → opens the Research tab with the search pre-filled (e.g. "Winner's Meats" in Ohio SOS). If it's a "Cross-check" → highlights the related document in the rail.
Top bar
What you click
What happens
15
Research
Navigates to Research tab. Shows all external source searches with their status and results. See Research tab breakdown below.
16
Financials
Navigates to Financials tab. Year-over-year 990 data, revenue trend, 990-T restaurant data. See Financials tab breakdown below.
17
Timeline
Navigates to Timeline tab. Chronological view of all events in the case. See Timeline tab breakdown below.
18
Generate referral package
Opens a pre-export checklist showing confirmed findings, linked documents, and entities. One more click to generate the PDF. Cannot generate if no findings are Confirmed.
19
Stat block (e.g. "4 Critical")
Same as clicking the Critical filter chip — filters the center panel to show only that severity.
Research tab — click map
Research
Pull data from IRS, Ohio SOS, AOS, county recorder, parcel search. Each source has its own search form and result set.
What you click
What happens
A
Source selector (IRS / SOS / AOS / Recorder / Parcel)
Switches the search form. Each source has different input fields (EIN vs. name vs. county + parcel number).
B
Search / Submit button
For slow sources (IRS, AOS, Parcel): queues an async job, shows a spinner, polls every 2s. For fast sources (SOS, Recorder): returns inline. Status pill shows Queued → Running → Done.
C
Result card
Expands to show full detail (all fields returned from the source). Collapsed state shows name + type + one key fact.
D
"Add to case" on a result
Creates an entity (person, org, or property depending on result type) in the case. Shows a confirmation: "Karen Homan added as Person." Finding suggestions appear if the entity matches an existing finding pattern.
E
Recent searches (sidebar or list)
Re-displays the result from a previous search in this case. Does not re-run the search — shows the cached result with the date it was pulled.
F
Job status pill (Queued / Running / Failed)
Running: just shows progress. Failed: shows the error + a Retry button. Clicking Retry re-queues the job.
Financials
Year-over-year 990 data, revenue trend, 990-T restaurant income/loss. Data pulled from IRS TEOS.
What you click
What happens
A
Year row (e.g. "2022")
Expands that year to show the full financial breakdown: Part I totals, Part IV checklist answers, Part VI governance fields, Part VII compensation. Collapses others.
B
Financial line item (e.g. "Revenue $4.5M")
Shows the source: which 990 document, which XML field, which line on the form. If a finding is connected to this line item, shows a badge linking to it.
C
Chart bar / data point
Highlights that year in the table. If an SR-021 (revenue spike) finding exists for that year, highlights the finding badge alongside it.
D
"990-T" sub-tab
Switches to restaurant UBIT view: gross receipts, cost of goods, salaries, NOL carryforward by year. Same click behaviors as main 990 table.
E
"Fetch 990 data" button
Runs the IRS TEOS fetch for this case's EIN. Queues an async job. On completion, populates or refreshes the financial table and re-evaluates financial signal rules.
F
Governance field (e.g. "COI Policy: No")
If a finding is tied to this field (e.g. SR-012), shows a badge. Clicking the badge jumps to that finding in the main case view.
Timeline
Chronological view of every event in the case — LLC formations, deeds, UCC filings, 990 filings, permits, referrals. The investigation story in order.
What you click
What happens
A
Event on the timeline
Opens a detail panel: the event date, source document, entities involved, and any findings connected to this event. Does not navigate away — panel slides in alongside the timeline.
B
Event type filter (Deeds / 990s / UCC / SOS / Permits)
Hides/shows event types. Lets you isolate, e.g., only property transactions to see the full acquisition sequence in order.
C
Finding badge on an event
Jumps back to the main case view with that finding selected. The badge indicates "this event is part of a confirmed finding."
D
Source document link on an event
Navigates to the main case view, switches to Document mode, opens that document. Back button returns to the timeline.
E
"Add event manually" button
Lets you add an event that isn't in a document — e.g. "Correction memo sent to FCA OIG, March 15 2026." Takes a date, description, and optional document attachment.
