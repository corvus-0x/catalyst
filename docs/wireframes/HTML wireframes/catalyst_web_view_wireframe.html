Colored dot badges on each knot show thread count: 3 red = active threads still pulling, 2 green = confirmed findings. Click any knot or any edge dot to see its detail.

The Investigate tab has four drill-down levels. Each level zooms in on one thing. "Back" always takes you up one level.

Web view — Investigate tab default
All knots (persons + orgs) as nodes. All threads as edges. Thread state encoded in edge color + style. Badge on each knot shows active vs. confirmed thread counts. Entry point for all investigation navigation.
Left rail: flat list of all knots with thread counts. Right panel: case-level stats + Sonnet case overview.
Knot view — click a node in web view
Focuses on one person or org. Shows all threads attached to this knot (including threads that connect to other knots). Lists documents that mention this entity. Sonnet shows per-knot observations.
Header: "← Back to web." Left rail: this knot's threads. Center: thread list with statuses. Right: Sonnet suggestions for this knot.
Thread view — click an edge dot, or "Pull this thread"
The active investigation workspace. Shows the thread question, all evidence gathered so far (from multiple documents), Sonnet's automatic "what to pull next" suggestions, and the tie-off options. This is where investigation work actually happens.
Header: "← Back to knot [name]." Left rail: documents in this thread + RAG search. Right: Sonnet suggestions, auto-refreshed when evidence added.
Document view — click a document in thread view or left rail
Full document facsimile. Findings linked to this doc are highlighted inline. Right panel shows which threads this document feeds into. RAG search sidebar available. Haiku extraction candidates shown for review.
Header: "← Back to thread [name]." "Add finding" always visible. Highlight text → right-click → Search other docs.
At any level, the top navigation bar shows the breadcrumb: Web → Karen Homan → Hidden compensation → 990-EZ 2021

Every interactive element on the web view.

Click target
Outcome
1
Click a knot node (person or org shape)
Navigates to knot view for that entity — center panel shows all threads attached to it, right panel shows Sonnet per-knot observations
2
Click a thread dot (colored circle on an edge)
Opens thread detail in right panel — title, status, evidence count, signal rules, documents. "Pull this thread" button takes you to full thread view.
3
Click a thread line (anywhere on the edge itself)
Same as clicking the dot — selects that thread. The wide invisible hit area makes this easy to click.
4
"+ Add knot" button (top toolbar)
Opens a picker: search existing case entities to promote to a knot, or create a new person/org. Placed on the web canvas at a default position; can be dragged.
5
"+ Add thread" button (top toolbar)
Click mode activates: click one knot, then another to draw a thread between them. Thread question drawer opens to name the thread.
6
Knot in left rail list
Highlights that knot in the SVG web; same as clicking the node. Does not navigate away from web view.
7
Drag a knot node
Repositions the node on the canvas. Layout is persistent — if you arrange the web to match how you think about the case, it stays that way.
8
Scroll / pinch on web canvas
Zooms in/out. "Fit all" button in toolbar resets to full case view.
9
Right-click a knot
Context menu: "Open knot view / Add thread from here / Mark as secondary knot / Remove from web"
10
Badge on knot (red = active, green = confirmed count)
Shows a tooltip with a mini-list of thread names. Does not navigate — just informational at a glance.
